객체(Object)

        - 속성과 기능으로 이루어진 것
            -> 속성 == 변수(필드)
            -> 기능 == 메서드(객체에 작성된 함수)

        - {} (블록) 내에 Key, Value가 모여있는 형태
            -> Key == 변수명/메서드명 (중복x)
            -> Valeu == 값, 함수 

        **** K:V 가 모여있는 형식 == Map (맵) ****

        - JS 객체 작성법    
        {K:V, K:V, K:V, ...}

        [JS 객체 생성 방법]
        1. {} : JS 객체 리터럴 표기법을 이용한 생성

        2. 생성자 함수 : 생성자 + new 생성자()를 이용한 생성

        ** 리터럴 == 값 **
        - value 대신에 리터럴이라는 명칭을 사용

        ex)
        문자열(string) -> "",''
        숫자(number) -> 1, 2.34, +50, -50
        배열(array) -> []
        객체(object) -> {}
        논리(bollean) -> true, false

생성자(Constructor) 함수

        * 생성자란?(== 생성자 함수)
        - 객체 생성 시 수행되는 기능(함수)

        -특정한 모양의 객체가 다수 필요한 경우
            이를 만들기 위한 코드의 중복을 최소화
            + 재사용성 증가
        * new 연산자 : (메모리에) 객체를 새롭게 생성(할달)
        
        * 생성자 함수를 이용한 객체 생성 방법
        -> const 객체명 = new 생성자함수();

JSON

        - JavaScript Object Notation
        (자바 스크립트 객체 표기법)
        - 자바스크립트 객체 모양으로 작성된 문자열(string)

            -> string은 모든 언어에서 공통으로 지원하는 데이터 타입
            -> 다른 프로그램과 데이터 교환 시 string을 주로 사용
                => 다량의 데이터를 가볍고 간단히 작성할 수 있는
                string 모양으로 JS 객체를 활용

        JS     객체 :    {"멤버아이디" : "user01", "멤버비밀번호" : "pass01", "나이" : 20}
        JSON 문자열 : '  {"멤버아이디" : "user01", "멤버비밀번호" : "pass01", "나이" : 20}  '
        - 간단한 포멧
        괄호 {} 내에서 key : value 쌍으로 구성 {"key" : value}
        key : 반드시 문자열 사용(쌍 따옴표("") 표기 필수)
        value : String, Boolean, Number, Array, Object, null 데이터 저장

        - 객체 {} 또는 배열[] 데이터를 효율적으로 표시 가능

        - 경량 데이터 교환 방식

        - 간단한 포맥을 가지고 있어 이해하기 쉬움

        - 순수 TEXT 기반
        구조화된 TEXT 형식
        대부분의 프로그래밍 언어에서 JSON 포멧 데이터를 핸들링 할 수 있는 라이브러리를
        시트템간 객체 교환에 용이

JS에서 제공하는 JSON 내장 객체

        - JSON (JS 객체 형태의 문자열)을 다루기 위한 객체 제공

        - JSON.stringify(JS 객체) : JS 객체 => JSON 문자열로 변환

        - JSON.parse(JS 객체) : JSON 문자열 -> JS 객체 변환
