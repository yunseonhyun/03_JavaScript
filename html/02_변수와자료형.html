변수(variable)

        * 메모리에 값을 계속 바꿔가면서 저장할 수 있는 공간

        * 변수에 이름을 붙이고 원하는 값을 대입하면
          필요한 곳에서 변수명을 호출하여 저장된 값을 사용할 수 있다.
        
          
변수명 명명 규칙

          - 의미 있는 이름을 작성 (대충 a, b, 20250722, minsu_login 이런식으로 짓는 것을 지양)
          - 대, 소문자 구분
          - 특수 문자는 _(언더스코어)만 사용 가능 (단, 첫 글자 제외)
          - 숫자도 사용 가능 (단, 첫 글자 제외)
          - 카멜 표기법으로 작성 loginShopping

          [카멜 표기법(Camel Case)]
          문장, 여러 단어를 변수명으로 작성할 때 이어지는 첫 번째 단어를 대문자로 작성
          ex) user name -> userName

javjaScript 변수의 종류와 특징

        var : 변수, 변수명 중복 o(덮어쓰기), 함수 레벨에서 scope
        let : 변수, 변수명 중봉 x, 블록{} 레벨 scope
        const : 상수, 변수명 중봉 x, 블록{} 레벨 scope

        1순위 : const (JS는 특정 요소를 선택해서 사용하는 경우가 많아서 변수에 고정시켜둠)

        2순위 : let
        
        3순위 : var(ES6 이후부터 사용 빈도가 많이 적어짐)

자바스크립트의 자료형

        자바스크립트는 변수 선언 시 별도의 자료형을 지정하지 않음
        -> 변수에 대입되는 값(리터럴)에 의해서 자료형이 결정됨

        - string(문자열)
        - number (숫자) -> 정수/실수 모두 포함
        - boolean (논리값) -> true(참), false(거짓)
        - object(객체) -> 배열 (Arrat), 자바스크립트 객체 {k:v, k:v}
        - function (함수)
        - undefined (정의되지 않은 변수) -> 변수가 선언만 됐을 뿐, 값이 존재하지 않는 변수이름

        (참고)
        - null (참조하는게 없음)
            -> document.getElementById("오타") (가장 흔한 경우)
            -> id가 일치하는 요소가 없어서 참조할 수 없다.
            (null은 타입보다 값(리터럴)의 개념)
